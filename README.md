# 📖 Welcome

Welcome to **The Linux Handbook** — an open-source, hands-on learning platform designed to help you **master Linux from first principles**.

Whether you're a:

- 🎓 Student
- 💻 Software Developer
- ☁️ Cloud Engineer
- ⚙️ DevOps Engineer
- 🔐 Cyber Security Professional
- 🖥️ System Administrator

this handbook is designed to help you build a **strong, practical, and industry-ready understanding of Linux**.

Unlike traditional Linux courses that focus on memorizing commands, this repository focuses on understanding **how Linux works**, **why it works that way**, and **how professionals use it every day**.

---

# 🎯 Why This Handbook?

Most Linux resources look something like this:

```
pwd
ls
cd
mkdir
cp
mv
rm
```

Students memorize commands.

A week later...

Most of them forget.

---

This handbook follows a different philosophy.

Instead of asking:

> **"Which command should I memorize?"**

we ask:

> **"What problem am I trying to solve?"**

Once you understand the problem, the command becomes easy to remember.

You'll learn:

- Why Linux works the way it does
- What happens behind the scenes
- How Linux is used in production environments
- How these concepts apply to Cyber Security, Cloud Computing and DevOps

Every module in this handbook follows the same five-step learning process.

```
📖 Learn
      ↓
🧪 Practice
      ↓
🧩 Challenge
      ↓
📝 Revise
      ↓
🚀 Apply
```

---

# 🚀 Getting Started

Getting started with **The Linux Handbook** is simple.

## 1️⃣ Clone the Repository

```bash
git clone https://github.com/Jayan-Shah/Linux-Course.git
```

Move into the project directory.

```bash
cd Linux-Course
```

---

## 2️⃣ Setup Your Learning Environment

Choose any one of the supported environments.

| Environment | Recommended |
|-------------|-------------|
| 🐧 Native Linux | ⭐⭐⭐⭐⭐ |
| 🪟 Windows (WSL2) | ⭐⭐⭐⭐⭐ |
| 🐳 Docker | ⭐⭐⭐⭐⭐ |
| ☁️ AWS EC2 | ⭐⭐⭐⭐ |
| 💻 Virtual Machine | ⭐⭐⭐⭐ |

Complete the setup by following:

```text
00-Setup/
```

---

## 3️⃣ Follow the Learning Journey

Do **not** jump between modules.

Complete them in order.

```text
00 → 01 → 02 → 03 → ...
```

Every module builds upon the previous one.

---

## 4️⃣ Learn the Right Way

For every module follow this workflow.

```text
📖 Read the Theory
        │
        ▼
💻 Try the Practical Examples
        │
        ▼
🧩 Solve the Mission
        │
        ▼
📝 Revise using the Cheat Sheet
        │
        ▼
🧠 Complete the Quiz
        │
        ▼
🚀 Move to the Next Module
```

Following this process will help you understand Linux instead of memorizing commands.
