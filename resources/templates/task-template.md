# 🧩 Linux Task Template

> **Purpose:** This template defines the standard structure for every hands-on task in the Linux Course. Every task should encourage exploration, problem-solving, and practical learning rather than memorization.

---

# 📌 Task Information

| Field            | Description                                 |
| ---------------- | ------------------------------------------- |
| Task Number      | e.g. Task 01                                |
| Task Name        | Short descriptive name                      |
| Module           | Module this task belongs to                 |
| Difficulty       | 🟢 Beginner / 🟡 Intermediate / 🔴 Advanced |
| Estimated Time   | e.g. 10–20 Minutes                          |
| Concepts Covered | Linux concepts practiced                    |

---

# 🎯 Mission Briefing

Start with a story.

Instead of saying:

> Create a directory.

Say something like:

> You have just logged into a Linux server for the first time.
>
> The system administrator has asked you to locate an important project before deployment.
>
> Can you find it?

Students should feel like they're solving a real problem.

---

# 📚 Learning Objectives

By completing this task, learners should:

* Practice concept 1
* Practice concept 2
* Practice concept 3

---

# 📖 Scenario

Explain the situation.

Examples:

* Production server
* Web server
* Cloud VM
* Developer workstation
* Compromised machine
* File server

Make it realistic.

---

# 🛠️ Objective

Clearly state what the learner needs to accomplish.

Example:

* Find a hidden file.
* Navigate to a directory.
* Read a configuration file.
* Search for logs.
* Create a backup.

The objective should never reveal the solution.

---

# 📜 Rules

Clearly define the rules.

Example:

* Don't modify existing files.
* Don't use GUI tools.
* Use only Linux commands learned so far.
* Internet search is allowed.
* Read the documentation if required.

---

# 📂 Initial Directory Structure

Provide the directory layout.

Example:

```text
project/

├── logs/

├── backup/

├── documents/

└── secret.txt
```

Students immediately understand the environment.

---

# 💻 Expected Commands

List the commands learners are expected to use.

Example:

* pwd
* ls
* cd

Avoid commands they haven't learned yet.

---

# 💡 Hints

Hints should be progressive.

### Hint 1

Very small clue.

---

### Hint 2

A stronger clue.

---

### Hint 3

Almost gives the answer.

Students should still think.

---

# ✅ Validation

Explain how learners know they've solved the task.

Examples:

* A success message appears.
* A validation script prints "Congratulations".
* A flag is revealed.
* A file is found.
* The expected output matches.

Every task should have a clear success condition.

---

# 🏆 Expected Outcome

Describe what success looks like.

Example:

```
Congratulations!

You found the project directory.
```

---

# 📖 Explanation

After solving the task, explain:

* Why the solution worked.
* Which Linux concepts were used.
* What happened behind the scenes.

Learning happens after solving.

---

# 🔐 Cyber Security Insight

Connect the task to Cyber Security.

Examples:

* File enumeration
* Log analysis
* Incident response
* Privilege escalation
* Malware investigation

Help learners understand where these skills are used professionally.

---

# 💼 Real World Connection

Explain where this task appears in industry.

Examples:

* Linux Administration
* DevOps
* Cloud Computing
* Production Support
* Security Operations

---

# 🧠 Reflection Questions

Ask learners to think.

Examples:

1. Why did this command work?
2. Could another command solve the same problem?
3. What would happen if the file were hidden?
4. How would this look on a production server?

Reflection builds deeper understanding.

---

# 📚 Learning Outcome

After completing this task, learners should be able to confidently:

* Skill 1
* Skill 2
* Skill 3

---

# ✅ Task Quality Checklist

Before publishing a task, verify:

* [ ] Story-based mission.
* [ ] Clear objective.
* [ ] Realistic scenario.
* [ ] Appropriate difficulty.
* [ ] Only previously learned commands are required.
* [ ] Progressive hints included.
* [ ] Validation exists.
* [ ] Cyber Security relevance explained.
* [ ] Industry connection included.
* [ ] Reflection questions added.
* [ ] Learning outcomes documented.

If every checkbox is complete, the task is ready to publish.
