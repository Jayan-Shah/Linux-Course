# 📄 Cheat Sheet Template

> **Purpose:** This template defines the standard structure for every module cheat sheet. A cheat sheet is **not** a replacement for the chapter—it is a quick revision guide that learners can use during practice, labs, or interview preparation.

---

# 📌 Cheat Sheet Information

| Field        | Description                                 |
| ------------ | ------------------------------------------- |
| Module       | Module Name                                 |
| Difficulty   | 🟢 Beginner / 🟡 Intermediate / 🔴 Advanced |
| Reading Time | 2–5 Minutes                                 |

---

# 🎯 Goal

The purpose of this cheat sheet is to help learners:

* Quickly revise important concepts.
* Recall commonly used commands.
* Remember syntax.
* Avoid common mistakes.
* Prepare for interviews.

---

# 📚 Key Concepts

Summarize the most important concepts in a few bullet points.

Example:

* Every Linux path starts from `/`.
* Linux uses a hierarchical filesystem.
* Everything in Linux is treated as a file.

Keep explanations short.

---

# 💻 Important Commands

| Command | Purpose                         |
| ------- | ------------------------------- |
| `pwd`   | Print current working directory |
| `ls`    | List directory contents         |
| `cd`    | Change directory                |

Only include commands taught in the current module.

---

# ⚙️ Common Options

| Command | Option | Description              |
| ------- | ------ | ------------------------ |
| `ls`    | `-a`   | Show hidden files        |
| `ls`    | `-l`   | Long listing format      |
| `cd`    | `..`   | Move to parent directory |

---

# 📂 Common Paths

Include important directories or paths.

Example:

| Path       | Purpose             |
| ---------- | ------------------- |
| `/home`    | User files          |
| `/etc`     | Configuration files |
| `/var/log` | System logs         |

---

# ⚠️ Common Mistakes

Examples:

❌ Confusing `/` with `/root`

❌ Deleting files from `/tmp` expecting them to remain permanently

❌ Using relative paths without checking the current directory

Explain the correct approach.

---

# 💡 Pro Tips

Examples:

* Use `Tab` for auto-completion.
* Use `history` to view previous commands.
* Use the Up Arrow to reuse commands.
* Learn to read command help using `--help` or `man`.

These tips improve productivity.

---

# 🔐 Cyber Security Notes

Mention any security-related points.

Examples:

* Check `/var/log` during investigations.
* Avoid storing sensitive data in `/tmp`.
* Configuration files inside `/etc` are valuable during enumeration.

---

# 🎤 Interview Questions

Include a few common interview questions.

Examples:

1. What is the Root Directory?
2. What is the difference between `/` and `/root`?
3. Why are hidden files used?
4. Explain absolute and relative paths.

---

# 📝 One-Minute Revision

End with a short section learners can read in under a minute.

Example:

* Linux has one Root Directory.
* Use `pwd` to know where you are.
* Use `ls` to see what's around you.
* Use `cd` to move between directories.
* Absolute paths always begin with `/`.

This section should help learners refresh the module quickly before interviews or practical sessions.
