# 📝 Quiz Template

> **Purpose:** This template defines the standard structure for every module quiz in the Linux Course. The goal is to evaluate understanding, encourage critical thinking, and prepare learners for real-world scenarios rather than simply testing memorization.

---

# 📌 Quiz Information

| Field          | Description                                 |
| -------------- | ------------------------------------------- |
| Module         | Module Name                                 |
| Difficulty     | 🟢 Beginner / 🟡 Intermediate / 🔴 Advanced |
| Questions      | Total Number of Questions                   |
| Estimated Time | e.g. 10–15 Minutes                          |

---

# 🎯 Quiz Objectives

After completing this quiz, learners should be able to:

* Validate their understanding.
* Identify weak areas.
* Apply Linux concepts.
* Think like a Linux Engineer.

---

# 🧠 Section 1 — Conceptual Understanding

Test whether the learner understands **why** something exists.

Example Questions:

1. Why does Linux use a single Root Directory?
2. What is the difference between the Shell and the Kernel?
3. Why are hidden files useful?
4. Why do Linux systems organize files into directories?
5. What is the purpose of `/etc`?

These questions should test understanding—not memorization.

---

# 💻 Section 2 — Command-Based Questions

Present a task and ask which command should be used.

Example:

### Question

You want to display your current working directory.

Which command would you use?

* A. ls
* B. pwd
* C. cd
* D. cat

---

### Question

You want to list all hidden files.

Which command should you execute?

* A. ls
* B. ls -a
* C. pwd
* D. cd

---

# 🌍 Section 3 — Real World Scenarios

Give learners practical situations.

Example:

> You've logged into a production Linux server.
>
> The application has stopped working.
>
> Where would you begin your investigation?

These questions encourage problem-solving.

---

# 🔐 Section 4 — Cyber Security Thinking

Every module should include Cyber Security questions.

Example:

1. Which directory would you inspect for system logs?
2. Why might attackers abuse `/tmp`?
3. Why should configuration files inside `/etc` be protected?
4. What information can be gathered from `/home`?

Learners begin thinking like defenders and penetration testers.

---

# ⭐ Bonus Challenge

Include one question that requires exploration.

Example:

> Open your Linux terminal.

Run a command that lists hidden files.

How many hidden files exist inside your Home Directory?

This encourages learners to use Linux instead of guessing.

---

# ✅ Answer Key

Provide answers at the end.

Format:

| Question | Answer |
| -------- | ------ |
| Q1       | B      |
| Q2       | C      |
| Q3       | A      |

Whenever possible, explain **why** the answer is correct.

# 💡 Revision Tips

If learners score poorly, recommend revisiting:

* Specific chapter(s)
* Hands-on exercises
* Missions
* Cheat Sheet

Encourage learning, not just scoring.
